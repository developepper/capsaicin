# T11: Prompt assembly

**Phase**: 4 — Adapter Layer
**Type**: Implementation

## Goal

Build implementer and reviewer prompts from ticket context.

## Scope

- `src/capsaicin/prompts.py`
- `build_implementer_prompt(ticket, criteria, prior_findings, cycle,
  max_cycles) -> str`
- `build_reviewer_prompt(ticket, criteria, diff_context, prior_findings) -> str`
- Implementer prompt elements: role instruction, ticket title/description,
  criteria with statuses, prior findings (when revising), cycle info, scope
  constraint
- Reviewer prompt elements: independent reviewer role instruction, diff, ticket
  title/description, criteria, prior findings with dispositions, JSON schema for
  expected output (full Review Result Schema from `adapters.md`), anti-bias
  instruction (do not trust commit messages or inline rationale)
- Tests: prompts contain all required elements; prior findings included only
  when non-empty

## Non-goals

- No template customization
- No adapter-specific formatting

## Acceptance criteria

- Implementer prompt contains all elements from `cli.md:121-127`
- Reviewer prompt contains all elements from `cli.md:161-167`
- Reviewer prompt includes full JSON schema
- Reviewer prompt includes anti-bias instruction
- Prior findings omitted when empty list
