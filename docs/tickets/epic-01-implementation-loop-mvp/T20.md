# T20: `capsaicin ticket review`

**Phase**: 6 — Review Pipeline
**Type**: Implementation
**Risk**: Heaviest integration ticket — touches 6 prior modules

## Goal

Full review pipeline integrating drift check, adapter invocation, validation, AC
updates, and finding reconciliation.

## Scope

- `capsaicin ticket review [TICKET_ID] [--allow-drift]`
- Find ticket in `in-review`
- Workspace drift check (T16): reject unless `--allow-drift`
- Capture review baseline (T16)
- Update orchestrator state
- Assemble reviewer RunRequest (prompt builder from T11)
- Insert `agent_runs` with `exit_status='running'`
- Invoke reviewer adapter (T13)
- Update `agent_runs` with terminal status
- Post-review baseline violation check (T16): if violation ->
  `contract_violation`, discard findings, increment review retry or block
- Parse/validate review result (T17): if failure -> `parse_error`, increment
  review retry or block
- Valid `fail`: reconcile findings (T19), update AC (T18), transition to
  `revise`
- Valid `pass` (high/medium confidence): reconcile (bulk-close), update AC,
  transition to `human-gate` with `gate_reason='review_passed'`
- Valid `pass` (low confidence): same as above but
  `gate_reason='low_confidence_pass'`
- Valid `escalate`: transition to `human-gate` with
  `gate_reason='reviewer_escalated'`
- Cycle-limit hit: prefer `human-gate` with `gate_reason='cycle_limit'`
- Update orchestrator state appropriately
- Log events to `activity.log`

## Non-goals

- No loop automation
- No human-gate commands

## Acceptance criteria

- Drift rejected without `--allow-drift`
- Drift accepted with `--allow-drift` and diff re-captured
- Contract violation (reviewer modified files) detected and handled
- `parse_error` handled with retry/block
- Valid `fail`: findings persisted, AC updated, ticket -> `revise`
- Valid `pass`: prior findings bulk-closed, AC updated, ticket -> `human-gate`
- Low-confidence pass -> `gate_reason='low_confidence_pass'`
- Escalate -> `gate_reason='reviewer_escalated'`
- Review retry limit exceeded -> `blocked`
- All orchestrator/state_transition records correct
