# T10: Adapter contract and types

**Phase**: 4 — Adapter Layer
**Type**: Implementation
**Status**: Complete

## Goal

Define RunRequest, RunResult, ReviewResult types and base adapter interface.

## Scope

- `src/capsaicin/adapters/__init__.py`
- `src/capsaicin/adapters/types.py` — dataclasses:
  - `RunRequest` (run_id, role, mode, working_directory, prompt, diff_context,
    context_files, acceptance_criteria, prior_findings, timeout_seconds,
    adapter_config)
  - `RunResult` (run_id, exit_status, duration_seconds, raw_stdout, raw_stderr,
    structured_result, adapter_metadata)
  - `ReviewResult` (verdict, confidence, findings, scope_reviewed)
  - `Finding` (severity, category, location, acceptance_criterion_id,
    description, disposition)
  - `ScopeReviewed` (files_examined, tests_run, criteria_checked)
  - `CriterionChecked` (criterion_id, description)
  - `AcceptanceCriterion` (id, description, status) — for run request context
- `src/capsaicin/adapters/base.py` — `BaseAdapter` ABC with
  `execute(request: RunRequest) -> RunResult`
- JSON serialization/deserialization for all types (dataclass -> dict -> JSON,
  and reverse)
- Tests: round-trip serialization; type validation

## Non-goals

- No concrete adapter implementation
- No prompt assembly

## Acceptance criteria

- RunRequest can represent implementer and reviewer invocations
- RunResult carries optional structured_result as ReviewResult
- ReviewResult enforces verdict/confidence/findings constraints at type level
- All types round-trip through JSON cleanly
