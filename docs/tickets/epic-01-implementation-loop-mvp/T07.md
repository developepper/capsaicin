# T07: `capsaicin ticket dep`

**Phase**: 2 — Project & Ticket Management
**Type**: Implementation

## Goal

Add ticket dependencies with cycle detection.

## Scope

- `capsaicin ticket dep TICKET_ID --on DEPENDENCY_ID`
- Validate both tickets exist
- Reject self-dependency (CHECK constraint handles this, but validate at CLI
  level too for error message)
- Detect cycles via DFS/BFS before writing
- Insert dependency edge
- Print confirmation

## Non-goals

- No dependency visualization
- No bulk import

## Acceptance criteria

- Valid dependency inserted
- Nonexistent ticket IDs error clearly
- Self-dependency rejected
- Circular dependency detected and rejected (A->B->C->A)
- Duplicate dependency handled gracefully (idempotent or clear error)
