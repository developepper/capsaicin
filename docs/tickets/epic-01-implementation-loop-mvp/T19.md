# T19: Finding reconciliation

**Phase**: 6 — Review Pipeline
**Type**: Implementation

## Goal

Reconcile findings across review cycles using fingerprinting.

## Scope

- `src/capsaicin/reconciliation.py`
- `compute_fingerprint(category, location, description) -> str` —
  `(category, location, description_prefix)` where prefix = first 80 chars,
  lowercased, whitespace-collapsed
- `reconcile_findings(conn, ticket_id, review_run_id, impl_run_id,
  new_findings, verdict, is_first_cycle)`:
  - First cycle: persist all as new with generated ULIDs
  - `verdict:pass`: bulk-close all prior open findings with
    `resolved_in_run=impl_run_id`
  - `verdict:fail`: match by fingerprint; update matched (description,
    severity); close unmatched prior; create unmatched new
- Tests: each reconciliation path

## Non-goals

- No semantic matching
- No human disposition overrides

## Acceptance criteria

- Fingerprint deterministic for same inputs
- Fingerprints differ when description prefix differs (same category/location)
- First cycle: all findings persisted as new with ULIDs
- Pass verdict: all prior open findings bulk-closed
- Fail verdict: matched findings updated; unmatched prior closed; unmatched new
  created
- Matched findings preserve original ID
