# T05: `capsaicin init`

**Phase**: 2 — Project & Ticket Management
**Type**: Implementation
**Risk**: First integration point (DB + config + filesystem)

## Goal

Initialize a project with directory structure, database, config, and initial DB
rows.

## Scope

- `capsaicin init [--project NAME] [--repo PATH]`
- Create `.capsaicin/projects/<slug>/` directory tree (including `renders/`,
  `exports/`)
- Slug generation: lowercase, hyphens for spaces, strip special chars
- Create `capsaicin.db`, run migrations
- Write default `config.toml` with resolved absolute `repo_path`
- Insert `projects` row with config snapshot in `projects.config`
- Insert `orchestrator_state` row with `status = 'idle'`
- Create empty `activity.log`
- Log `PROJECT_INIT` event to `activity.log` using the locked format:
  `<ISO8601> PROJECT_INIT project_id=<id> {}`
- Error if project slug already exists

## Non-goals

- No ticket commands
- No adapter validation

## Acceptance criteria

- Directory structure matches `architecture.md` layout
- `capsaicin.db` exists with all tables
- `config.toml` exists with correct defaults and absolute `repo_path`
- `projects` row exists in DB with config snapshot
- `orchestrator_state` row exists with `idle` status
- `activity.log` exists with init event in locked format
- Re-running init for the same project errors cleanly

## Dependencies

- T02
- T03
- T04

## Notes

Define a small `activity_log.py` helper here (or inline) for the append-only log
format. This helper will be reused by later tickets. Format:
`<ISO8601> <EVENT_TYPE> [project_id=X] [ticket_id=X] [run_id=X] <JSON>`.

This is the first integration point. Expect at least one round of fixes to
T02/T03/T04 interfaces.
