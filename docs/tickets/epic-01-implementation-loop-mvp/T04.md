# T04: Config loading

**Phase**: 1 — Foundation
**Type**: Implementation

## Goal

Parse `config.toml`, validate required fields, return typed config.

## Scope

- `src/capsaicin/config.py`
- `load_config(config_path) -> Config` — dataclass with typed fields
- Validate required sections: `[project]`, `[adapters.implementer]`,
  `[adapters.reviewer]`, `[limits]`
- Apply defaults for optional fields (`timeout_seconds=300`,
  `ticket_selection.order="created_at"`, etc.)
- `resolve_project(capsaicin_root) -> str` — auto-resolve if exactly one project
  exists; error on zero or multiple
- `write_default_config(path, project_name, repo_path)` — write the default
  config.toml from `configuration.md`
- Tests: missing sections raise clear errors; defaults applied; project
  resolution logic

## Non-goals

- No DB snapshot sync (that happens in init/command entrypoints)
- No adapter-specific config validation

## Acceptance criteria

- Valid config.toml parses into structured Config
- Missing required sections raise clear errors with section name
- Defaults are applied for omitted optional fields
- Single-project auto-resolution works; multiple projects error without explicit
  selection
