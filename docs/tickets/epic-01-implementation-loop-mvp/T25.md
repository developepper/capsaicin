# T25: `capsaicin status`

**Phase**: 8 — Status & Recovery
**Type**: Implementation

## Goal

Render project and ticket status to stdout.

## Scope

- `capsaicin status [--ticket TICKET_ID] [--verbose]`
- Without `--ticket`: project summary
  - Ticket counts by status
  - Active ticket from `orchestrator_state`
  - Tickets in `human-gate` with `gate_reason`
  - Blocked tickets with `blocked_reason`
  - Next runnable `ready` ticket (deps satisfied, ordered by `created_at`)
- With `--ticket`: ticket detail
  - Title, status, `status_changed_at`, cycle/attempt info
  - Acceptance criteria with statuses
  - Open findings grouped by severity
  - Last run summary (role, exit_status, duration, verdict)
- With `--verbose`: add full run history and transition history
- All output to stdout, formatted for terminal readability

## Non-goals

- No rendered file output
- No color beyond basic formatting

## Acceptance criteria

- Project summary shows correct counts per status
- Active ticket displayed when one exists
- Human-gate tickets show `gate_reason`
- Blocked tickets show `blocked_reason`
- Next runnable ticket identified correctly (respects deps)
- Ticket detail shows all specified fields
- Verbose mode includes run and transition history
