# T06: `capsaicin ticket add`

**Phase**: 2 — Project & Ticket Management
**Type**: Implementation
**Status**: Complete

## Goal

Create tickets with acceptance criteria via CLI args or TOML file.

## Scope

- `capsaicin ticket add --title TITLE --description DESC [--criteria "..."]`
  (repeatable `--criteria`)
- `capsaicin ticket add --from FILE` (TOML format per `cli.md`)
- Generate ULID for ticket and each criterion via `python-ulid`
- Insert ticket with `status = 'ready'`
- Insert acceptance criteria with `status = 'pending'`
- Record state transition (`null -> ready`, triggered_by `human`)
- Log `TICKET_CREATED` event to `activity.log`
- Print ticket ID and brief summary to stdout

## Non-goals

- No dependency management
- No rendered file output (stdout only)
- No batch import

## Acceptance criteria

- Inline creation produces correct ticket + criteria rows
- File import parses TOML format and creates ticket + criteria
- Criteria are FK-linked to ticket
- State transition row recorded
- Ticket ID printed to stdout
- Missing required fields (title, description) error clearly
- ULID format is used for IDs

## Dependencies

- T05

## Notes

Use the `python-ulid` package for ID generation. ULIDs are preferred over UUIDs
per data-model.md because they are sortable and more readable in logs.
