# T22: `capsaicin ticket revise`

**Phase**: 7 — Human Gate & Decisions
**Type**: Implementation
**Suggested PR pairing**: Ship with T21, T23, T24 (all human-gate commands)

## Goal

Human-initiated revision from human-gate.

## Scope

- `capsaicin ticket revise [TICKET_ID] [--add-finding DESCRIPTION]
  [--reset-cycles]`
- Find ticket in `human-gate`
- If `--add-finding`: create a synthetic `agent_runs` row with all required
  fields:
  - `id`: generated ULID
  - `ticket_id`: the current ticket
  - `role`: `'human'`
  - `mode`: `'read-write'`
  - `cycle_number`: the ticket's `current_cycle`
  - `attempt_number`: `1`
  - `exit_status`: `'success'`
  - `verdict`: `null`
  - `prompt`: `'human feedback via ticket revise'` (static sentinel)
  - `run_request`: `'{}'` (empty JSON object — no adapter invocation)
  - `started_at`: current ISO 8601 timestamp
  - `finished_at`: same as `started_at`
  - `duration_seconds`: `0.0`
  - all other fields (`raw_stdout`, `raw_stderr`, `structured_result`,
    `adapter_metadata`, `diff_context`): `null`
  Attach finding to this run with `severity='blocking'`,
  `category='human_feedback'`
- Record `revise` decision
- Transition to `revise`
- If `--reset-cycles`: reset cycle and retry counters
- Set orchestrator status to `idle`

## Non-goals

- No automatic re-implementation

## Acceptance criteria

- Revise transitions `human-gate -> revise`
- Human findings persisted with correct fields, attached to synthetic human
  `agent_run`
- Synthetic `agent_run` row satisfies all NOT NULL constraints in `agent_runs`
  schema (role, mode, cycle_number, exit_status, prompt, run_request,
  started_at)
- Decision row recorded
- `--reset-cycles` resets counters; without it, counters preserved
- Orchestrator -> `idle`
