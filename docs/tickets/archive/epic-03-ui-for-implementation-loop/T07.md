# T07: Finish packaging, tests, and operator documentation for the UI

**Phase**: 6 — Packaging And Docs
**Type**: Implementation
**Status**: Planned

## Goal

Complete the UI feature as a packaged, documented, and tested local operator
surface.

## Scope

- ensure templates and static assets are included in package builds
- add test coverage for:
  - `capsaicin ui`
  - key routes
  - representative action flows
  - SSE behavior at an appropriate level
- update top-level and docs-based operator documentation for the UI
- document any new dependencies and local runtime assumptions
- update roadmap references once the epic is underway or complete

## Non-goals

- no visual-polish-only work detached from behavior
- no browser-E2E suite unless needed to cover a gap not reachable through ASGI
  tests

## Acceptance criteria

- a packaged install includes everything needed to run the UI locally
- automated tests cover the core launch and operator flows with deterministic
  local fixtures
- docs explain how to launch and use the UI without requiring code inspection
- the tickets and roadmap docs point to the implemented UI path clearly
