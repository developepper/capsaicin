# T05: `capsaicin doctor`

**Phase**: 3 — Preflight Checks
**Type**: Implementation
**Status**: Complete

## Goal

Expose preflight validation through a single operator command that can be run
before expensive agent work.

## Scope

- add `capsaicin doctor`
- render check results from T04 in a concise terminal format
- include actionable failure text and exact remediation guidance where possible
- exit non-zero when one or more failing checks are present
- include warnings without failing the command when appropriate
- for missing Claude write permissions, report the exact missing entries:
  `Edit` and `Write`

## Non-goals

- no automatic permission repair
- no background monitoring

## Acceptance criteria

- `capsaicin doctor` validates command availability, repo path, git repo state,
  working tree cleanliness, and Claude permission configuration
- failing checks cause a non-zero exit
- warnings are rendered distinctly from failures
- output tells the operator what to fix rather than only what is wrong
- missing Claude write permissions tell the operator to add `Edit` and `Write`
  to `<repo>/.claude/settings.local.json` under `permissions.allow`

## Implementation Notes

- Reuse T04 check helpers directly; do not duplicate environment validation in
  the CLI command.
- Keep the output terminal-friendly and scannable. A short checklist-style
  format is preferable to paragraphs.
- Exit status should map to check severity consistently: failures non-zero,
  warnings still visible, all-pass zero.
- Include exact remediation text for missing Claude permissions and invalid repo
  setup. Avoid vague messages.
- Keep the command focused on validation only. No auto-fix behavior in this
  ticket.
