# T07: Status and human-gate diagnostic visibility

**Phase**: 4 — Loop And Status UX
**Type**: Implementation
**Status**: Complete

## Goal

Expose the new reliability diagnostics in `status` and human-gate inspection so
operators can understand what happened without reading raw envelopes.

## Scope

- extend `capsaicin status --ticket` and `--verbose` output
- update status queries to fetch the fields needed for diagnostics, including
  adapter metadata and persisted raw envelope text where required
- show last-run outcome details, including permission-denied summaries and
  useful agent result text where appropriate
- extract agent result text from persisted `raw_stdout` at display time rather
  than requiring a new column
- show cost in last-run summaries when available
- keep default output concise and reserve expanded diagnostics for verbose views
- ensure the previous misleading case of `exit=success` plus
  `gate_reason=empty_implementation` is no longer opaque in verbose output

## Non-goals

- no UI work
- no long-form rendered reports

## Acceptance criteria

- ticket status output shows high-signal diagnostic details for the most recent
  run
- verbose output exposes richer last-run metadata without requiring database
  inspection
- permission-denied and empty-implementation cases are distinguishable in
  status output
- last-run summaries can surface cost and concise agent remediation text when
  those signals explain the outcome

## Implementation Notes

- Use the real misleading status case from manual validation as the regression
  target: `exit=success` plus `gate_reason=empty_implementation` must no longer
  hide permission denials, cost, or short remediation text.
- Keep default `status` output compact. Put richer fields in `--verbose` rather
  than turning the default view into a log dump.
- Reuse the same normalized denial and outcome information introduced by T01-T03
  so `status` reflects authoritative persisted state.
- T02 owns inline stop messaging and human-gate context rendering. This ticket
  owns retrospective `capsaicin status` views.
- Favor stable, parseable terminal wording over decorative formatting. This
  command is likely to feed both humans and test snapshots.
