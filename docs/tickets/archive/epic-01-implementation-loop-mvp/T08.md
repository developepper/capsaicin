# T08: State machine module

**Phase**: 3 — State Machine & Orchestrator
**Type**: Implementation
**Status**: Complete

## Goal

Enforce ticket transition rules as a reusable module.

## Scope

- `src/capsaicin/state_machine.py`
- `LEGAL_TRANSITIONS` — dict/set encoding all transitions from
  `state-machine.md`, keyed by `(from_status, to_status)` with allowed actors
- `transition_is_legal(from_status, to_status, actor) -> bool`
- `transition_ticket(conn, ticket_id, to_status, triggered_by, reason=None,
  gate_reason=None, blocked_reason=None)`:
  - Validate legality
  - Update `tickets.status`, `status_changed_at`, `gate_reason`,
    `blocked_reason`
  - Insert `state_transitions` row
- Dependency satisfaction guard for `ready -> implementing`: query
  `ticket_dependencies` and check all deps are `done`
- Log `STATE_TRANSITION` event to `activity.log`
- Tests: every legal transition accepted; `revise -> pr-ready` rejected;
  `ready -> implementing` with unmet deps rejected; guard conditions set correct
  `gate_reason`/`blocked_reason`

## Non-goals

- No orchestrator state updates (T09)
- No cycle/retry counter management

## Acceptance criteria

- All legal transitions from `state-machine.md` accepted
- All illegal transitions rejected (especially `revise -> pr-ready`, direct to
  `pr-ready` without `human-gate`)
- `ready -> implementing` with unmet dependencies rejected
- Each transition creates a `state_transitions` row
- `gate_reason` and `blocked_reason` set correctly per guard conditions
