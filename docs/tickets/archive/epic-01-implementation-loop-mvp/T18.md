# T18: Acceptance criteria updates

**Phase**: 6 — Review Pipeline
**Type**: Implementation

## Goal

Update criterion statuses based on review results.

## Scope

- `src/capsaicin/criteria.py`
- `update_criteria_from_review(conn, ticket_id, review_result: ReviewResult)`
- Logic per `cli.md:189-200`:
  - Match `criteria_checked` entries by `criterion_id`
  - Match findings by `acceptance_criterion_id`
  - Checked + blocking finding -> `unmet`
  - Checked + no blocking finding -> `met`
  - Not checked -> unchanged
- Tests: various combinations of checked/unchecked criteria with/without
  blocking findings

## Non-goals

- No finding persistence
- No reconciliation

## Acceptance criteria

- Criterion marked `met` when checked with no blocking finding
- Criterion marked `unmet` when checked with blocking finding
- Criterion unchanged when not checked
- Null `acceptance_criterion_id` findings don't affect criteria
- Multiple criteria updated correctly in one pass
