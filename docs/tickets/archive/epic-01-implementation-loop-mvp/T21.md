# T21: `capsaicin ticket approve`

**Phase**: 7 — Human Gate & Decisions
**Type**: Implementation
**Suggested PR pairing**: Ship with T22, T23, T24 (all human-gate commands)

## Goal

Human approval gate with workspace verification.

## Scope

- `capsaicin ticket approve [TICKET_ID] [--rationale TEXT] [--force]`
- Find ticket in `human-gate`
- Verify workspace matches reviewed diff; reject unless `--force`
- Require rationale when `gate_reason` is `cycle_limit`,
  `reviewer_escalated`, or `low_confidence_pass`
- Record `approve` decision with rationale
- Transition to `pr-ready`
- Set orchestrator status to `idle`
- Print PR preparation summary to stdout (ticket title, acceptance criteria
  statuses, finding summary)

## Non-goals

- No PR creation
- No rendered file output

## Acceptance criteria

- Approval succeeds when workspace matches and gate_reason is `review_passed`
- Rationale required for `cycle_limit`/`reviewer_escalated`/`low_confidence_pass`
- Missing rationale for those gate_reasons errors
- Workspace drift rejects without `--force`
- `--force` overrides drift check
- Decision row recorded
- Ticket -> `pr-ready`
- Orchestrator -> `idle`

## Dependencies

- T08
- T09
- T14

## Notes

`pr-ready` is a terminal human-handoff state in MVP. PR creation and merge
remain manual.
