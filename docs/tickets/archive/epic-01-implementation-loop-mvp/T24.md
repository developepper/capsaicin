# T24: `capsaicin ticket unblock`

**Phase**: 8 — Status & Recovery
**Type**: Implementation
**Suggested PR pairing**: Ship with T21, T22, T23 (all human-gate commands)

## Goal

Return a blocked ticket to `ready` for another attempt.

## Scope

- `capsaicin ticket unblock TICKET_ID [--reset-cycles]`
- Accept only from `blocked`
- Record `unblock` decision
- Clear `blocked_reason`
- Transition to `ready`
- If `--reset-cycles`: reset cycle and retry counters
- Set orchestrator status to `idle`
- Log `TICKET_UNBLOCK` event to `activity.log`

## Non-goals

- No automatic re-run

## Acceptance criteria

- Blocked ticket -> `ready`
- Decision row recorded with `decision='unblock'`
- `blocked_reason` cleared
- `--reset-cycles` resets counters; without it, preserved
- Non-blocked tickets rejected
- Orchestrator -> `idle`
