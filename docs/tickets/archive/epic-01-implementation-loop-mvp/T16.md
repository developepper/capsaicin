# T16: Review baseline and workspace drift check

**Phase**: 6 — Review Pipeline
**Type**: Implementation
**Suggested PR pairing**: Ship with T14 in a single PR (T16 extends T14)

## Goal

Capture review baselines and enforce workspace-drift policy.

## Scope

- `src/capsaicin/review_baseline.py`
- `check_workspace_drift(conn, repo_path, run_id) -> DriftResult` — compare
  current `git diff HEAD` against `run_diffs.diff_text`
- `capture_review_baseline(conn, repo_path, run_id)` — snapshot pre-review
  tracked-file state into `review_baselines`
- `check_review_violation(conn, repo_path, run_id) -> bool` — compare
  post-review state to baseline
- `handle_drift(conn, run_id, repo_path, allow_drift: bool)` — if drift and
  allow_drift=True, re-capture `run_diffs`; if drift and allow_drift=False,
  raise
- Tests: temporary git repos with controlled modifications

## Non-goals

- No review invocation
- No result validation

## Acceptance criteria

- Drift detected when workspace differs from captured implementation diff
- No-drift case passes cleanly
- `--allow-drift` re-captures diff as new baseline
- Review violation detected when reviewer modifies tracked files
- No-violation case passes cleanly
- `review_baselines` rows created and compared correctly

- T14
