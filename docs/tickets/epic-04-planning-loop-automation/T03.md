# T03: Planning Commands, Queries, And CLI Surface

## Goal

Add the shared command/query services and operator CLI surface for creating,
inspecting, and manually driving the planning loop.

## Scope

- add app command services for planning mutations
- add app query services and read models for planning views
- add CLI commands for starting planning, drafting, reviewing, revising,
  approving, deferring, and showing planning status
- add human-readable renderers for planning summaries and detail views
- ensure project resolution and config refresh work the same way as the
  existing ticket loop

## Non-goals

- automatic multi-step planning loop execution
- UI pages
- GitHub export
- implementation-ticket materialization

## Acceptance Criteria

- operators can create a planning record from a problem statement without
  editing the DB directly
- operators can inspect planning status at both summary and detail levels
- manual commands exist for planner draft/review/revise/approve flows in the
  same style as `ticket run`, `ticket review`, and `ticket approve`
- shared app commands/queries are the source for both CLI and later UI usage
- CLI output makes planning state, gate reasons, blocked reasons, and active
  findings easy to inspect
- tests cover command resolution, invalid state handling, and status rendering
- the CLI namespace is explicit and documented. Default expectation:
  `capsaicin plan new`, `capsaicin plan draft`, `capsaicin plan review`,
  `capsaicin plan revise`, `capsaicin plan approve`, `capsaicin plan defer`,
  and `capsaicin plan status`

## Implementation Notes

- Preserve the existing narrow, pragmatic CLI shape. If new verbs are added,
  they should feel parallel to the ticket loop rather than like a separate
  product.
- Prefer a `plan` command namespace over scattering planning verbs at the top
  level.
- Manual operation should remain possible even after loop automation exists.
  The CLI should support stepping the planning workflow one phase at a time.
- T04 is expected to add `capsaicin plan loop`; this ticket should establish
  the manual stepping commands that loop builds on.
