# T02: Add the local web runtime and `capsaicin ui`

**Phase**: 2 — Web Runtime
**Type**: Implementation
**Status**: Planned

## Goal

Add the minimal local web runtime needed to host the operator UI without
introducing a separate frontend toolchain.

## Scope

- add a Python ASGI app inside the existing package
- add `capsaicin ui`
- bind to `127.0.0.1` only
- choose an available port automatically unless `--port` is provided
- support `--no-open` to suppress browser launch
- serve templates and static assets from the Python package
- wire route handlers to the shared services and read models from T01
- add basic request-level error handling suitable for local operator use
- establish safe SQLite request handling for mixed CLI/UI local usage

## Non-goals

- no main dashboard content beyond a basic shell if later tickets own it
- no SSE yet
- no JSON API design beyond what the runtime strictly needs

## Acceptance criteria

- `capsaicin ui` starts a local server successfully against an initialized
  project
- the command can auto-open the browser and can also run with `--no-open`
- the server resolves project context using the same rules as the CLI
- route handlers invoke shared Python services directly rather than shelling
  out to CLI commands
- templates and static assets are packaged with the project cleanly
- the web runtime keeps SQLite transactions short and explicitly documents or
  configures its concurrency assumptions

## Implementation Notes

- Prefer Starlette unless a concrete requirement appears that makes FastAPI the
  better fit.
- Keep the first route layer thin and boring. This ticket is plumbing, not UX.
- Be explicit about lifecycle and connection ownership so SQLite usage remains
  predictable under local HTTP requests.
- Review whether `get_connection()` should enable WAL mode for local mixed
  CLI/UI usage. If WAL is adopted, do it deliberately and document the reason.
- Even if WAL is not adopted in this ticket, document the concurrency model and
  keep web handlers out of long-lived transactions.
- Browser auto-open should be best-effort and must not be required for the
  command to succeed.
- Add only the dependencies needed for a durable server-rendered setup.
- Read T01's resulting service and read-model contracts first. This ticket
  should consume them, not redefine them.
