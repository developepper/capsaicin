# T03: Build dashboard and queue read models

**Phase**: 3 — Read Models And Screens
**Type**: Implementation
**Status**: Planned

## Goal

Create the dashboard data model and first operator screen for project overview,
queue state, and active work.

## Scope

- refine the initial dashboard read-model contract introduced in T01 so it
  cleanly supports a real screen and includes:
  - project summary
  - orchestrator state
  - active ticket/run
  - human-gate inbox summary
  - blocked summary
  - next runnable ticket
  - recent activity or recent runs summary
- implement the main dashboard route and template
- support HTMX-driven partial refresh for high-signal sections where useful
- keep the screen centered on the current implementation loop

## Non-goals

- no ticket detail screen
- no human-gate action form design beyond lightweight links or placeholders
- no live updates yet

## Acceptance criteria

- the dashboard renders the current project state without requiring terminal
  output
- queue and gate state are legible and derived from shared read models rather
  than route-local query assembly
- the screen exposes enough context to decide what to open next
- the template remains server-rendered and works without custom frontend build
  tooling

- this ticket may include UI triggers for `run`, `review`, `resume`, or `loop`

## Implementation Notes

- Resist the temptation to dump every field onto the dashboard. Optimize for
  operator decisions, not schema completeness.
- The dashboard should answer: what is active, what is blocked, what needs
  human action, and what runs next.
- If a partial refresh is added, keep the endpoint shape aligned with the same
  read model rather than creating a second query path just for HTMX.
- Include a clear path from dashboard state to next operator action, but do not
  collapse dashboard and ticket detail into one overloaded page.
- If action buttons for `run`, `review`, `resume`, or `loop` are introduced
  here, treat them as thin wrappers over shared command services and redirect
  immediately to a page that can show progress.
